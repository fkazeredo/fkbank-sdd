# Sprint 1 — release notes (draft)

Draft prepared at Sprint closure. This is **not** a release: no version is chosen here, and nothing
is tagged or published. Publication is a separate, owner-controlled `/release` decision that may
reuse this draft.

- **Integrated candidate:** `e12711b`
- **Sprint Assurance:** `SECURITY_OBSERVATIONS` — every applicable mandatory control executed;
  `SECURITY_VERIFIED` is not claimed because branch protection is not configured (owner-accepted
  risk `SEC-0001`).

## Goal

Walking skeleton observable from day one, the accounting heart proven by race tests, and a person
opening an account and signing in.

## Delivered capabilities

- **Sign-up and account opening.** A person signs up, passes automated KYC against the bureau
  emulator, and gets an account opened with a materialized `$0.00` balance. No zero-value ledger
  posting is created. Same-CPF sign-ups converge
  to exactly one customer; sign-up is idempotent by CPF while a check is pending.
- **Sign-in.** OIDC + PKCE sign-in behind an embedded Authorization Server, landing on a zero
  balance for the new account.
- **Ledger core.** Append-only double-entry postings, four-decimal money rounded only at the edge,
  balances that cannot go negative, reversal-at-most-once by contra-posting, and a trial balance
  that audits stored balances against the postings.
- **Observability baseline.** Per-request correlation id propagated into async work and every
  structured JSON log line, an authorization-failures counter on a token-protected metrics
  endpoint, and a drift-checked public OpenAPI document.
- **Walking skeleton.** Three-root architecture with ArchUnit and Modulith gates, default-deny
  authorization, Flyway over PostgreSQL, an Angular shell behind a single published edge, an
  end-to-end stack, and CI.

## User-visible changes

- First usable journey: sign up → KYC → account opened → sign in → see a zero balance, in a real
  browser.
- A rejected sign-up now shows the specific reason a `422` carries, with the offending field
  flagged inline.

## Operational and deployment changes

- A single published edge fronts the Angular shell and the API and sets the security headers (CSP,
  anti-clickjacking, `nosniff`, `Permissions-Policy`).
- PostgreSQL with Flyway migrations; the bureau emulator (`emulators/bureau`) is a test double, not
  a shipped artifact.

## Known limitations (required before any production, end-user-facing release)

- Raw CPF can reach application logs through a database-driver error on the duplicate-CPF path, in
  the production profile (`SEC-F-01`, CWE-532).
- The coming-of-age timezone is unsettled across layers; the leap-day rule is enforced server-side
  only, so a 29-February birth date can block the browser form on a birthday eve.
- E-mail uniqueness is unenforced while an application is pending, and a stranded pending
  application has no automatic recovery or expiry path.
- Branch protection is not configured on the repository (`SEC-0001`, owner-accepted).
